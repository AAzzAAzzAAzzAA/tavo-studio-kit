# Tavo 其他功能总结

## 1. 这一组是什么

`其他` 页面本身不是单独的一个复杂系统，而是一个索引页。它把一些相对独立、偏工具型的功能归到一起，方便从设置页进入。

官方索引页列出的子功能共有 5 个：

- 存储空间
- 备份与恢复
- 图片发送
- 自定义聊天快捷键
- 群聊快捷发言

下面按子功能分别总结。

## 2. 存储空间

### 作用

用于查看 Tavo 当前占用的存储空间，并进行可安全清理的数据清理。

### 路径

- 打开左侧边栏
- 点击“更多”
- 进入“设置”
- 打开“存储空间”

### 页面里能看到什么

- Tavo 已用空间总量
- 可清理数据分类

文档明确列出的可清理项有：

- 缓存：例如 TTS 语音缓存，清掉后再次播放需要重新请求 API
- 日志：上下文日志、负载均衡日志等
- 角色：例如角色头像、图片等资源

### 重要限制

文档特别强调：

- “数据”类别不提供清理选项
- 也就是聊天、角色、世界书等核心数据不会在这里被直接清除

这意味着这个页面更像“安全释放空间”，不是“重置资料”。

## 3. 备份与恢复

### 作用

用于备份核心数据，并支持通过备份文件恢复。

### 路径

- 打开左侧边栏
- 点击“更多”
- 进入“设置”
- 打开“备份与恢复”

### 核心能力

- 创建完整数据备份
- 备份时可选择是否包含 API 密钥
- 设置备份提醒
- 通过备份文件恢复数据

### 最重要的注意事项

文档强调了版本兼容性问题：

- 高版本创建的备份，不能在更低版本里恢复

所以恢复前应尽量保证：

- 当前应用版本不低于备份创建时的版本

这是后续做迁移、归档、设备切换时必须记住的一点。

## 4. 图片发送

### 作用

启用后，可以向角色发送图片，由配置的多模态端点生成图片描述，再把描述融入聊天。Tavo 本地并不“自己识图”，识别质量和可用性取决于所选模型/端点。

### 路径

文档写的路径是：

- 左侧边栏
- 更多
- 语音
- 聊天设置
- 图片发送

后续如果界面版本变化，可以把这个路径当作“以聊天设置中的图片发送页为准”的参考。

### 核心配置

#### 开关

- 先启用“图片发送”功能

#### 图片描述 API

这是最关键的一步。文档要求所选模型必须支持多模态识图能力。

文档举的例子包括：

- GPT-4V
- Claude 3
- Gemini

也就是说，纯文本模型不够，需要能看图的模型端点。端点里勾选 `image` 能力只代表 Tavo 会按多模态路径尝试调用，不证明上游模型一定支持；最终以实际请求结果为准。

#### 两类提示词

文档提到有两种提示词：

- 图片描述生成提示词
- 图片描述注入提示词

官方建议：

- 如果不熟悉提示词编写，先用默认值

### 适合的用途

- 让角色识图并回应
- 图像剧情互动
- 多模态角色陪伴

## 5. 自定义聊天快捷键

### 作用

用于创建一组个人常用短语，聊天时一键填充到输入框中。

### 路径

- 左侧边栏
- 更多
- 设置
- 聊天设置
- 自定义聊天快捷键

### 可做的事

- 添加快捷条目
- 为条目设置标签和实际内容
- 对单个条目启用/禁用

### 生效方式

- 在聊天界面调出键盘后
- 启用的快捷键会出现在键盘上方或附近
- 点击后会直接把对应内容填入输入框

### 适合的用途

- 常用开场白
- 固定命令
- 常见剧情动作
- 角色控制短语

## 6. 群聊快捷发言

### 作用

在群聊里快速指定”由谁来回复”，避免频繁手打 `@角色名` 或其他指定发言方式。

### 路径

- 左侧边栏
- 更多
- 设置
- 聊天设置
- 群聊快捷发言

### 启用后的界面变化

- 输入框左上角会显示当前群聊成员头像按钮

### 使用方式

1. 先在输入框写好你想表达的内容
2. 直接点击目标角色头像
3. Tavo 会尝试触发该角色对你的输入做出回复；实际是否生成成功仍取决于群聊设置、当前端点和请求状态

### 群聊提示词系统

群聊在预设层面有专门的提示词支持：

| 预设项 | 用途 |
|--------|------|
| `preset_prompt_group_chat_start` | 群聊开场的提示词模板 |
| `preset_prompt_group_chat_start_hint` | 开场提示词的占位提示文本 |
| `new_group_chat_prompt` | 新建群聊时的默认系统提示词 |

### 群聊相关设置

- `setup.mig.misc.show.group.chat.name` — 群聊名称显示开关（在聊天界面显示群聊标题）
- `misc_settings_show_group_chat_name` — UI 设置项：是否显示群聊名称
- `misc_settings_show_group_chat_shortcuts` — UI 设置项：群聊快捷发言按钮

### 群聊相关宏

- `{{group}}` — 群聊中所有角色的名字列表
- `{{charIfNotGroup}}` — 非群聊时显示当前角色名，群聊时为空
- `{{groupNotMuted}}` — 群聊中未被禁言的角色列表

### 适合的用途

- 多角色群聊控制
- 快速点名发言
- 降低群聊操作成本

## 7. 备份与恢复（技术细节补充）

### 备份 API 方法

Tavo 内部使用 `bata_backup_*` 方法族处理备份逻辑（`bata` 前缀来自 Tavo 内部命名）：

| 内部方法 | 功能 |
|----------|------|
| `bata_backup_create_backup` | 创建备份文件 |
| `bata_backup_backup_title` | 备份文件的标题/名称 |
| `bata_backup_backup_desc` | 备份文件的描述信息 |
| `bata_backup_backup_api_secret` | 备份中 API 密钥的处理逻辑 |
| `bata_backup_backup_success` | 备份成功回调 |
| `bata_backup_backup_failed` | 备份失败回调 |
| `bata_backup_reminder_action_backup` | 备份提醒触发时的操作 |

### 备份文件结构

备份文件内部包含 `backup_info.txt` 元数据文件，记录了：

- 备份创建时间
- 备份版本号
- 创建备份时的 Tavo 版本

### API 密钥与备份安全

- 备份时可**选择**是否包含 API 密钥（`endpoint_edit_api_secret`）
- 如果包含密钥，备份文件应视为敏感文件
- 分享角色卡或世界书时，备份文件不应随卡一起分发

### 版本兼容性

- 高版本创建的备份**不能**在更低版本恢复（向下不兼容）
- 恢复前应确保当前 Tavo 版本 ≥ 备份创建时的版本
- 版本校验失败会提示：”Backup failed, please check the backup file version”

## 8. 键盘快捷键与宏助手

### 键盘快捷键

Tavo 支持以下键盘快捷键配置：

| 设置项 | 功能 |
|--------|------|
| `enableFullscreenShortcut` | 全屏快捷键开关 |
| `enableMacroShortcut` | 宏快捷键开关 |
| `dbShortcutKeys` | 快捷键数据存储 |

### 自定义聊天快捷键

在聊天界面，自定义快捷键会出现在键盘上方或附近，点击后直接将对应内容填入输入框。适合：

- 常用开场白
- 固定命令
- 常见剧情动作
- 角色控制短语

### 宏助手

输入框左下角的”宏助手”按钮：

- 列出所有可用的变量宏
- 一键填入输入框
- 填入后可手动改变量名和数值
- 适合不熟悉宏语法的用户快速上手

## 9. Discord 社区

Tavo 有官方 Discord 社区（`app_discord`），带有 Discord logo 资源。相关讨论和支持可能在 Discord 上进行。

## 10. 这一组功能的共同特点

这几项都更偏”设置页工具能力”，而不是提示词层或剧情层能力。

可以把它们分成四类理解：

- 设备/数据管理：存储空间、备份与恢复
- 输入与交互效率：快捷键、群聊快捷发言、宏助手
- 多模态扩展：图片发送
- 社区与支持：Discord 社区

## 11. 后续做 Tavo 相关事情时值得记住的点

- 存储空间页能清缓存和资源，但不会清核心聊天/世界书数据
- 备份可能包含 API 密钥，做迁移时要注意安全
- 备份恢复受版本兼容性影响，高→低不可恢复
- 图片发送必须依赖多模态模型
- 快捷键适合做高频命令/常用句模板
- 群聊快捷发言适合做多角色控制型玩法
- 群聊有独立的提示词系统，不要用单聊预设直接套群聊
