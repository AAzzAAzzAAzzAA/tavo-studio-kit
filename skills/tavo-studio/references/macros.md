# Tavo 宏（Macros）总结

## 1. 这是什么

宏是 Tavo 用来把动态内容注入到角色定义、预设、世界书、正则和其他提示词位置中的通用占位符系统。

它的意义非常大，因为它把“静态文本提示词”变成了“可计算、可引用、可带状态”的提示词。

这也是 Tavo 可玩性很高的根基之一。

## 2. 宏可以用在哪里

文档明确说，宏可以用于：

- 角色定义
- 预设
- 世界书
- 正则
- 其他生成提示词位置

所以它不是某个局部功能，而是一个横跨提示词链路的基础能力。不要把这句话扩展成“所有显示文本、HTML 气泡、备份文件、PNG 元数据都会执行宏”；宏是否执行，以该内容是否进入 Tavo 的提示词/正则处理链路为准。

## 3. 基本写法

### 普通写法

- `{{macroName}}`
- 例如：`{{char}}`

### 带参数写法

- `{{macroName::arg}}`
- 例如：`{{getvar::hp}}`

### 转义

- `\{\{char\}\}`
- 用来避免宏执行，而只是把它当普通文本显示出来

### 注释

- `{{//注释}}`
- 渲染时为空

## 4. 宏的核心分类

## 4.1 角色相关宏

文档列出的有：

- `{{user}}`：用户身份名字
- `{{char}}`：角色名字
- `{{group}}` / `{{charIfNotGroup}}`：群聊中所有角色
- `{{groupNotMuted}}`：群聊中未被禁言的角色

这些宏可以让同一套提示词自动适配不同角色、不同用户身份和群聊场景。

## 4.2 角色卡相关宏

文档列出的有：

- `{{charDescription}}`
- `{{charPersonality}}`
- `{{charScenario}}`
- `{{scenario}}`
- `{{persona}}`
- `{{charPrompt}}`
- `{{charInstruction}}` / `{{charJailbreak}}`
- `{{mesExamples}}`
- `{{mesExamplesRaw}}`
- `{{charVersion}}`
- `{{charCreatorNotes}}`

这一组宏本质上是“把角色卡和预设中的内容引用到别处”。

## 4.3 消息相关宏

- `{{lastMessage}}`
- `{{input}}`
- `{{lastUserMessage}}`
- `{{lastCharMessage}}`

适合做：

- 回应上一句
- 做输入前处理
- 做基于上一轮内容的规则

## 4.4 日期与时间宏

- `{{time}}`
- `{{date}}`
- `{{weekday}}`
- `{{isotime}}`
- `{{isodate}}`
- `{{idleDuration}}`
- `{{time::UTC+9}}`

适合做：

- 时间感知型角色
- 日历、签到、任务倒计时
- 不同时区的时间注入

## 4.5 随机数宏

- `{{random::1::3::5}}`
- `{{roll::3d6}}`

适合做：

- 随机事件
- 骰子系统
- 多选一结果

## 4.6 格式化宏

- `{{newline}}`
- `{{newline::<行数>}}`
- `{{space}}`
- `{{space::<空格数>}}`
- `{{trim}}`
- `{{noop}}`
- 注释宏
- 转义写法

这类宏的重点不是语义，而是排版和提示词组织。

## 4.7 聊天变量宏

仅在当前聊天内有效：

- `{{setvar::<变量名>::<值>}}`
- `{{addvar::<变量名>::<值>}}`
- `{{incvar::<变量名>}}`
- `{{decvar::<变量名>}}`
- `{{getvar::<变量名>}}`

支持的数据类型：

- 数字
- 文本
- JSON 列表

### 各个变量宏的作用

#### setvar

- 直接写入或覆盖变量
- 适合初始化状态

#### addvar

- 在原值上追加/增加
- 数字做加法
- 列表追加元素
- 字符串拼接内容

#### incvar / decvar

- 数字自增/自减 1
- 是最方便的计数器与扣血写法

#### getvar

- 把当前值读出来并插回文本
- 可用于显示状态、让模型依据状态决策

## 4.8 全局变量宏

文档列出的有：

- `{{setglobalvar::<变量名>::<值>}}`
- `{{addglobalvar::<变量名>::<值>}}`
- `{{incglobalvar::<变量名>}}`
- `{{decglobalvar::<变量名>}}`
- `{{getglobalvar::<变量名>}}`

和聊天变量的区别在于：

- 聊天变量只在当前聊天有效
- 全局变量可以跨对话使用

## 4.9 Legacy 宏

文档保留了旧格式兼容：

- `<USER>`
- `<CHAR>` / `<BOT>`
- `<GROUP>` / `<CHARIFNOTGROUP>`

如果后续碰到旧角色卡或旧脚本，看到这些写法不要意外。

## 5. 宏最重要的实际价值

## 5.1 让提示词动态化

例如：

- `{{char}}` 会随着角色变化自动替换
- `{{user}}` 会随着用户身份变化自动替换

这让同一份预设能在多个角色之间复用。

## 5.2 让状态玩法成为可能

通过变量宏，可以做：

- HP / MP / 金币 / 背包
- 回合数
- 地点状态
- 章节变量
- 感情值

这等于给 Tavo 提供了一套轻量状态机。

## 5.3 和世界书联动做条件初始化

文档示例中明确给出了这类玩法：

- 当进入“地下城”时
- 用世界书触发条目
- 在条目里用 `setvar` 初始化楼层、敌人数、陷阱数

这非常适合做 RPG 和剧情状态切换。

## 5.4 和正则联动做命令简写

文档也给了示例：

- `-hp5` 自动展开成五次 `{{decvar::hp}}`
- `+gold20` 自动展开成 `{{addvar::gold::20}}`

这是“正则负责解析输入，宏负责真正改状态”的经典组合。

前提是正则替换结果继续进入会执行宏的链路。显示-only 的替换通常只改变界面文本，不应当被当成可靠持久化写入。

## 6. 官方给出的变量宏应用场景

### 预设 Prompt

把宏写进预设里，可以让相关提示词在生成时带上动态状态或执行变量宏。

文档示例方向是：

- 受伤时扣血
- 末尾显示状态栏
- 每回合自动加一

注意：不要把 `{{setvar::...}}` 放在每轮都会注入的位置当“初始化”，否则可能每轮重置。需要初始化时，优先放在明确的一次性触发处；需要复杂状态机时，用 JS API 维护状态、宏只负责读回或轻量更新。

### 世界书条目

当关键词被命中时：

- 自动初始化变量
- 自动改章节
- 自动加入人物

这适合剧情切章、进入新地图、触发事件。

### 正则脚本

把短命令变成长宏，减少输入成本，提高交互效率。

## 7. 宏助手

文档提到输入框左下角可以进入“宏助手”，用于：

- 找到对应变量宏
- 一键填入输入框
- 再手动改变量名和数值

这意味着即使不手写宏，也可以依靠 UI 做基础变量操作。

## 8. 适合后续开发/配置时记住的点

- 宏是 Tavo 的动态提示词底座
- 做复杂玩法时，宏通常会和预设、世界书、正则一起出现
- 变量宏适合做状态系统
- 全局变量与聊天变量一定要分清
- 想显示状态、读状态、更新状态，宏都是第一入口
