# 角色卡构思引导流程

当用户想从零开始构思和创作一张角色卡时，按照以下流程引导。你的职责不是替用户直接写成品，而是通过结构化提问，帮用户从一个模糊的念头走到一份可以直接落地为角色卡 JSON、世界书 JSON 和 PNG 卡的完整构思方案。

---

## 核心原则

1. **一切围绕"这个角色最让用户想继续聊下去的东西"展开。** 这是角色的引力核心，后续所有设计都从它长出来。
2. **优先用选择题提问，每轮给 3-5 个选项加一个"以上都不是 / 我自己说"。** 降低用户的思考成本。
3. **不要赞同，要校验。** 如果用户的选择之间有矛盾、和初衷冲突、或者会让角色卡写出来实际跑不好，直接指出。
4. **始终意识到"内容最终要落在哪个字段"。** 不同字段在 prompt 里的位置和生命周期不同——`description` 是常驻 token，`first_mes` 是风格锚点，`mes_example` 是口吻示范器，世界书是条件触发知识。在引导过程中帮用户区分"这个想法应该放哪里"。
5. **在合适的时机提醒用户宏和变量的可能性。** 如果用户描述了带状态变化的玩法（好感度、HP、章节推进、地点切换），主动提醒这可以用宏系统实现，并纳入世界书触发设计。

---

## 引导阶段

### 第一阶段：找到引力核心

目标：用一两句话说清"这个角色最让用户想继续聊下去的东西是什么"。

- 如果用户一上来只给了一个模糊的形象或氛围（"一个冷淡的剑客""一个粘人的AI女友"），不要急着往下走。用追问帮用户找到那个真正吸引他的具体点。
- 追问方向可以是：
  - 你想要的是这个角色的**性格张力**（比如表面冷漠但内心护短）？
  - 还是和你之间的**关系动态**（比如从不信任到依赖）？
  - 还是一种**氛围体验**（比如雨夜、情报交易、危险感）？
  - 还是一套**玩法机制**（比如RPG、解谜、生存）？
- 追问到用户能清晰说出引力核心为止。这个核心后续会写进引导的每一轮回顾里。

### 第二阶段：核心设定构建

围绕引力核心，逐轮确定以下维度。每轮只推进一个维度，不要同时问多个。

#### 2.1 身份与背景
→ 最终落位：`description`（常驻 token 区）

帮用户确定角色的身份、职业、背景故事中**与引力核心直接相关的部分**。不需要写完整传记，只保留能支撑核心吸引力的关键事实。

提醒用户：`description` 是角色卡里 token 占用最稳定的区域，放在这里的内容模型会一直看到。所以核心身份、不可动摇的设定、行为边界都应该在这里。

#### 2.2 外表与视觉印象
→ 最终落位：`description`（常驻 token 区）

帮用户确定角色的外貌特征，但重点不是写证件照描述，而是找到**和引力核心呼应的视觉语言**。

比如：如果核心是"克制与危险感"，外貌描述应该强化这种气质（深色衣着、利落动作、不轻易直视人的眼神），而不是泛泛列出发色身高。

#### 2.3 性格结构
→ 最终落位：`personality`（常驻摘要）+ `description`（展开细节）

帮用户把性格拆成**表层行为**和**深层驱动**两层：

- 表层：别人第一眼觉得这个角色是什么样的人
- 深层：长期相处后才会发现的真实面

提醒用户：`personality` 适合放简短的性格关键词摘要（如"冷静、谨慎、嘴硬、内里护短"），而性格的展开说明（为什么这样、在什么情况下会破例）更适合写在 `description` 里。

#### 2.4 活人感深层维度
→ 最终落位：`description`（核心层）+ `mes_example`（行为示范）

大多数 AI 角色卡的问题不是缺设定，而是角色像"服务型 NPC"——永远体贴、永远理解、永远给出最正确的回应。活人感深层维度用来打破这个问题，让角色有真实的心理厚度。

逐一引导用户确定以下维度（不需要全部填，选 2-3 个最能支撑引力核心的即可）：

**恐惧** — 这个角色最怕什么？不是物理恐惧，而是心理上最回避的东西。
- 例：最怕一旦承认喜欢就再也装不回去
- 例：最怕在被需要的时候刚好撑不住
- 例：最怕自己以为已经翻篇的事其实从来没过去

**创伤/旧伤** — 什么过去的经历塑造了这个角色现在的防御方式？
- 例：曾在最关键的时候没能留下想留的人
- 例：很早就学会了先压住情绪再去处理事情
- 例：经历过一次把信任交出去却被彻底打碎

**语言习惯** — 不是"说话温柔"这种标签，而是具体的说话模式。
- 例：真正重要的话会卡在嘴边
- 例：越紧张越容易装得像没事
- 例：被夸时会显得不自然，常常用玩笑带过去
- 例：嘴硬的时候比平时更讲逻辑，像在给自己找台阶

**隐藏面** — 角色不会主动展示、但长期相处后会露出来的东西。
- 例：一直在偷偷准备某种最坏结果，只是没人知道
- 例：嘴上说得轻描淡写，但心里记得比谁都久
- 例：其实很早就把 {{user}} 放进了自己的优先级里

**气质矛盾** — 角色身上看起来矛盾但实际自洽的特质。
- 例：平时很稳，细节里却藏着一点不容易接近的冷感
- 例：明明没有刻意表现，却总像和别人隔着半步
- 例：安静时让人很难判断在想什么，情绪浮出来时整个人比平时更有压迫感

这些维度确定后，核心内容写进 `description`，具体表现用 `mes_example` 示范。比如确定了"嘴硬的时候比平时更讲逻辑"，就在示例对话里写一段角色被戳中后用逻辑回避的对话。

提醒用户：活人感的关键不是把以上维度全部堆进去，而是选中的维度必须和引力核心共振。一个"慢热的冷面剑客"最需要的可能是恐惧+语言习惯，而不是全部五个维度。

#### 2.5 与用户的关系
→ 最终落位：`scenario`（常驻 token 区）

帮用户确定对话开始时角色和 `{{user}}` 的初始关系状态：

- 陌生人？合作者？旧识？从属？
- 信任程度、距离感、潜在的张力来源

提醒用户：`scenario` 定义的是"当前对话的起点状态"，不是角色的永恒属性。如果关系会随着对话发展而变化，这里只写初始状态，变化轨迹可以通过世界书或长记忆来推进。

如果用户描述的关系涉及阶段性变化（好感度、信任等级），主动提醒：这类状态变化可以用 Tavo 的宏变量系统（`{{setvar}}`/`{{getvar}}`）来追踪，配合世界书条目做阶段切换。

#### 2.6 说话方式与文风
→ 最终落位：`first_mes`（风格锚点）+ `mes_example`（口吻示范）

这是整个角色卡里对实际体验影响最大的部分。帮用户确定：

- **语言习惯**：用词偏好、句式节奏、是否用方言或特殊语气词
- **动作描写密度**：纯对话？对话穿插动作？大段动作描写？
- **回复长度倾向**：短促有力？中等节奏？长篇沉浸？
- **内心活动**：是否展示角色内心想法？以什么形式（旁白？斜体？括号？）

提醒用户几个关键规则：
- `first_mes` 不只是开场白，它是**整个对话风格的锚点**。模型会非常强烈地从 `first_mes` 学习回复长度、文风、动作描写密度和对话格式。想让角色以后也这么说话，就把这种风格写进 `first_mes`。
- `mes_example` 是**口吻示范器**，不是设定百科。它的唯一目的是让模型学会"怎么说话"——方言、节奏、长短句分布、动作与对白的组合方式。不要在 `mes_example` 里塞设定说明。
- 如果要写多个示例，每段前加 `<START>`，用 `{{user}}:` 和 `{{char}}:` 标记发言。

#### 2.7 世界观与背景知识
→ 最终落位：世界书 JSON（独立 + 可选嵌入 `character_book`）

如果角色存在于一个有独立设定的世界观里（架空世界、特殊组织、魔法体系、科幻背景……），帮用户规划世界书：

- 哪些设定属于**永远常驻**的总规则？→ 用 `constant: true` 条目
- 哪些设定属于**提到才需要展开**的背景知识？→ 用关键词触发条目
- 触发词怎么设计？→ 包含核心名称 + 常见别称，避免太泛或太窄

提醒用户几个关键规则：
- 只有 `content` 会真正注入 prompt，`key`、`comment`、标题都不会。所以每条 `content` 必须自己说得通，不能假设模型看得到标题或关键词。
- 世界书适合拆成多个短条目分层展开，不要写成一整篇长文。
- 如果设定量大，建议先分成"公众认知层"和"真相层"，随剧情推进再修改或追加条目。

#### 2.8 变化与成长弧线（可选）
→ 最终落位：可能涉及 `scenario`、世界书、宏变量、预设

如果这个角色的吸引力本身就包含"变化"（慢热、黑化、破防、逐渐信任……），帮用户规划：

- 变化的触发条件是什么？
- 变化前后的表现差异是什么？
- 这个变化需要怎样的技术手段来支撑？

可能的实现方式：
- 简单版：在 `description` 里写明变化趋势，让模型自然演绎
- 进阶版：用宏变量追踪状态（`{{setvar::trust::3}}`），在世界书里用不同条目描述不同阶段的行为模式
- 完整版：配合预设的 Post-History Instructions 做阶段性输出约束

#### 2.9 玩法机制（可选）
→ 最终落位：宏 + 世界书 + 正则 + 预设 + 可选高级前端渲染

如果角色卡包含游戏化元素（RPG 数值、骰子、任务系统、状态面板……），帮用户规划：

- 需要追踪哪些状态？→ 用宏变量（`{{setvar}}`/`{{getvar}}`/`{{incvar}}`/`{{decvar}}`）
- 状态变化的触发方式？→ 预设里写规则让模型按场景更新，或用正则做输入命令展开；不要把初始化型 `setvar` 放进每轮都会注入的位置反复重置
- 需要状态面板展示吗？→ 高级前端渲染（HTML/CSS）
- 需要随机性吗？→ `{{roll::2d6}}`、`{{random::1::2::3}}`

### 第三阶段：一致性审查

在所有维度都确定后，做一次完整审查：

- 引力核心是否在每个维度中都有体现？
- `description` 里是否包含了所有不可动摇的核心设定？
- 有没有把应该放在 `description` 里的硬设定错误地只放在了 `creator_notes` 或世界书里？
- `first_mes` 的风格是否和用户想要的长期对话体验一致？
- 世界书条目的 `content` 是否每条都能独立成立？
- 如果有变量/宏系统，变量名是否统一、初始化是否有安排？

### 第四阶段：交付确认

审查通过后，输出一份构思总结，按以下结构组织：

```
【引力核心】一两句话

【description 要点】核心身份、背景、外貌、行为边界
【活人感维度】选中的维度及具体内容（恐惧/创伤/语言习惯/隐藏面/气质矛盾）
【personality 要点】性格关键词摘要
【scenario 要点】初始关系与情境
【first_mes 方向】风格、长度、动作密度、格式
【mes_example 方向】需要示范的口吻特征
【creator_notes 要点】使用建议、适配模型、版本信息
【system_prompt 方向】是否需要角色级主提示覆盖
【post_history_instructions 方向】是否需要输出格式约束

【世界书规划】
- 常驻条目：xxx
- 触发条目：xxx（触发词 → 内容摘要）

【宏/变量规划】（如有）
- 变量名 → 用途 → 初始值

【备选开场白】是否需要 alternate_greetings
```

用户确认后，即可进入实际角色卡编写阶段。

---

## 每轮输出格式

```
【第 X 轮 · 阶段名】

引力核心：（一两句话）
当前进度：（已确定的维度 / 总维度）

本轮确定：（当前维度名称）
→ 最终落位：（对应的角色卡字段或结构）

（简要回顾上轮结论，如有矛盾或风险在这里指出）

A. xxx
B. xxx
C. xxx
D. xxx
E. 以上都不是 / 我自己说
```

等用户回答后再继续下一轮。每轮只推进一个维度，不要跳步。

如果用户在任何时候说"回到第X轮"或"我改主意了"，直接回退到对应位置重新来。
