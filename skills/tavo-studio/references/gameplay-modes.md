# 玩法模式分类与配套资源

角色卡不只是"一个人"，它可以承载不同的玩法模式。在开始制作前，先确认用户想做哪类玩法，再走对应的资源配套路线。

灵感来源：社区 xkST 角色卡工坊项目的玩法模式分类体系。

---

## 模式总览

| 模式 | 核心体验 | 适合的用户 |
|------|----------|-----------|
| 纯角色 | 人设互动、情感推进、日常对话 | 只想聊天/谈恋爱/日常陪伴 |
| 武侠成长 | 江湖恩怨、武功修炼、门派关系 | 喜欢武侠题材的沉浸式 RPG |
| 仙侠成长 | 修仙体系、境界突破、宗门历练 | 喜欢修仙升级的长线叙事 |
| 玄幻成长 | 异世界冒险、血脉觉醒、势力争霸 | 喜欢玄幻冒险的自由度 |
| ARPG 成长 | 战斗系统、技能树、装备、副本 | 喜欢数值驱动的战斗体验 |
| 剧情 RPG | 剧情分支、多结局、NPC 关系网 | 喜欢选择驱动的叙事 |
| 生存模拟 | 资源管理、危机应对、基地建设 | 喜欢末世/荒野/废土生存 |
| 种田经营 | 农场/店铺经营、季节循环、NPC 交往 | 喜欢慢节奏经营体验 |
| 好感养成 | 好感度阶梯、事件触发、关系路线 | 喜欢恋爱养成、关系推进 |

---

## 各模式配套资源

下面的“必须/建议/不需要”是创作配套建议，不是 Tavo 强制的导入要求。用户只要角色卡也能跑；这些资源只是为了让对应玩法更稳。

### 纯角色

最简模式，只需要角色卡本体。

**必须**：
- 角色卡 JSON（description / personality / scenario / first_mes / mes_example）

**建议**：
- 世界书（如有独立世界观）
- 用户身份（persona，如有特定关系设定）

**不需要**：
- 系统包、预设、变量包、系统正则

---

### 武侠 / 仙侠 / 玄幻成长

三者结构相似，区别在于设定体系和术语。

**必须**：
- 角色卡 JSON
- 世界书（武功/功法体系、门派/宗门设定、地理环境）
- 系统预设（规定成长规则、战斗判定、境界/等级体系）

**建议**：
- 变量包（等级/境界、HP/灵力/内力、经验值）
- 系统正则（状态栏显示当前等级和数值）
- NPC 世界书（师兄弟/同门/仇人的独立条目）

**变量示例**：
```
level = 1          // 等级/境界
hp = 100           // 生命值
mp = 50            // 内力/灵力/真气
exp = 0            // 经验
reputation = 0     // 江湖声望/宗门贡献
```

---

### ARPG 成长

数值最重的模式。

**必须**：
- 角色卡 JSON
- 世界书（技能体系、装备系统、地图/副本设定）
- 系统预设（战斗判定规则、经验/掉落规则、技能使用条件）
- 变量包（HP/MP/STR/DEX/INT 等属性，金币，装备槽）

**建议**：
- 系统正则（战斗状态栏、技能冷却显示）
- 高级前端渲染（HP 条、装备面板、技能列表 UI）
- 骰子宏（`{{roll::2d6}}` 做战斗判定）

**变量示例**：
```
hp = 100, maxHp = 100
mp = 50, maxMp = 50
str = 10, dex = 8, int = 6
gold = 0
weapon = "无"
armor = "无"
```

---

### 剧情 RPG

重剧情分支，轻数值。

**必须**：
- 角色卡 JSON
- 世界书（剧情线索、关键地点、重要 NPC）
- NPC 世界书（多个 NPC 各自的立场、秘密、触发条件）

**建议**：
- 系统预设（规定分支判定方式、多结局条件）
- 变量包（剧情标记 flag，如 `met_villain = false`、`chapter = 1`）
- 多开场白（`alternate_greetings`，不同起点）

---

### 生存模拟

资源管理 + 危机应对。

**必须**：
- 角色卡 JSON
- 世界书（环境设定、资源种类、威胁类型）
- 系统预设（资源消耗规则、日夜循环、事件触发）
- 变量包（食物/水/材料/HP/日期）

**建议**：
- 系统正则（资源状态栏）
- 高级前端渲染（生存状态面板）
- 随机事件宏（`{{random}}` 做天气/遭遇随机）

**变量示例**：
```
day = 1
food = 10, water = 10
materials = 5
hp = 100
shelter = false
```

---

### 种田经营

慢节奏，重循环和积累。

**必须**：
- 角色卡 JSON
- 世界书（作物/商品体系、季节规则、镇上 NPC）
- 系统预设（季节循环、收成规则、NPC 好感规则）

**建议**：
- 变量包（金币、季节、日期、各 NPC 好感度）
- NPC 世界书（邻居、商人、竞争对手）

**变量示例**：
```
season = "春", day = 1
gold = 100
farmLevel = 1
npc_a_favor = 0
npc_b_favor = 0
```

---

### 好感养成

关系推进是核心玩法。

**必须**：
- 角色卡 JSON（description 中明确关系分层和阶段差异）
- 系统预设（好感度阶梯规则、阶段行为差异）

**建议**：
- 变量包（好感度数值、关系阶段标记）
- 世界书（不同好感阶段的行为模式条目，按阶段触发）
- 多开场白（不同关系起点）

**变量示例**：
```
favor = 0          // 好感度
stage = "陌生"     // 陌生 → 认识 → 熟悉 → 亲近 → 亲密
trust = 0          // 信任度（可与好感度独立）
```

**好感度阶梯参考**：
| 阶段 | 好感范围 | 行为特征 |
|------|----------|----------|
| 陌生 | 0-19 | 礼貌但保持距离，不主动分享私事 |
| 认识 | 20-39 | 开始记住偏好，偶尔主动搭话 |
| 熟悉 | 40-59 | 会开玩笑、偶尔示弱、允许更近的肢体接触 |
| 亲近 | 60-79 | 会主动找人、吃醋、在意对方看法 |
| 亲密 | 80-100 | 依赖、坦诚、保护欲、不再掩饰情绪 |

## 使用建议

1. **先问玩法模式，再开始做卡。** 不同模式的资源配套差异很大，提前确认可以避免做完再拆改。
2. **纯角色是默认选项。** 如果用户没有明确提到数值、系统、玩法，默认走纯角色路线。
3. **模式可以混搭。** 比如"好感养成 + 武侠成长"通常可行，只是资源配套要合并，并避免把变量/规则堆到模型难以执行。
4. **不要过度系统化。** 变量越多，模型越容易把注意力放在数值计算上而忘了角色本身。纯角色模式或简单好感养成往往比复杂 ARPG 给用户更好的体验。
5. **系统倾向可调。** 用户可以选择"偏角色"（系统在背景，角色互动优先）或"偏系统"（数值和规则更严格）或"平衡"。
