# Tavo 预设总结

## 1. 这是什么

预设是一套给角色和对话环境使用的基础配置与行为模板。它的作用不是定义某个具体角色的全部资料，而是定义一组可复用的“提示词结构”和“行为规则”，让角色在不同聊天里都能按同一种风格工作。

可以把它理解成：

- 角色卡决定“这个角色是谁”
- 预设决定“系统要怎么组织提示词、怎么引导模型回复、怎么推进剧情”

对后续做 Tavo 相关功能时，预设通常是最重要的“系统层控制面”之一。

## 2. 预设解决什么问题

- 让角色创建不必每次从零写提示词
- 统一不同角色的对话风格、剧情推进方式和输出格式
- 给单聊、群聊、续写、AI 帮答等场景提供不同的系统引导
- 把“角色信息”“用户身份”“场景”“示例对话”“历史后指令”等内容拆开管理

## 3. 基础提示词

文档把下面这些视为预设中的“基础提示词”，它们主要负责构成角色和对话的基础框架。

### 用户身份

- 定义用户与角色之间的关系
- 会直接影响互动语气、亲密度、边界和称呼方式
- 适合放“你和角色是什么关系、角色应该如何看待你”

### 角色设定

- 定义角色的核心背景
- 包括职业、目标、历史、身份、立场等
- 更像角色的世界内身份卡片

### 性格特点

- 定义角色的行为倾向与情绪反应方式
- 适合写冷静/热情/固执/幽默/克制/支配欲等风格特征

### 情景设定

- 定义当前场景和环境
- 会直接影响回复中的氛围、动作、紧张程度和信息重点
- 比如“古老图书馆”“逃亡途中”“校园天台”等

### 新示例聊天

- 用示例对话告诉模型“你应该怎么跟用户聊”
- 比单纯写规则更直观
- 适合示范角色口吻、节奏、互动方式和典型回应

### 新聊天

- 设定新对话起始时的基础模式
- 用于开场状态、第一次互动的节奏和口吻

### 群聊推进

- 定义多角色对话时如何推动剧情
- 适合说明谁该主动发言、怎样把话题往前推、怎样让多人互动不僵住

### 继续推进

- 用于避免剧情停滞
- 适合放“在停顿时要主动推进情节”“要给出下一步行动”“要引出新的信息”

### AI 帮答（Impersonation）

- 用来控制 AI 在”解释、代答、补充说明”场景中的回答方式
- 更偏问答型辅助，而不是纯角色扮演

### Impersonation 技术细节

Impersonation（AI 帮答）除了 UI 级别的开关外，还涉及以下层面：

- **预设 prompt**：`preset_prompt_impersonation` — 控制 AI 帮答时的提示词模板
  - `preset_prompt_impersonation_hint` — 提示词模板的占位提示文本
- **内置 identifier**：`impersonation` — 在 PresetEntry 中使用此 identifier 可向帮答 prompt 中注入内容
- **运行时状态**：`impersonationLoading` / `impersonationTyping` — 帮答生成中的加载和打字状态
- **宏**：`{{charPrompt}}` — 可引用角色的主提示词；`{{charInstruction}}` / `{{charJailbreak}}` — 角色指令/防越狱提示

## 4. 其他详细提示词

这部分更像“精细化控制层”，适合在你已经有基本预设后继续做高精度调校。

### Main Prompt

- 总体指令
- 处于系统层的核心控制位
- 决定角色扮演深度、风格基调、剧情方向、优先级

### Word Info (Before)

- 对对话前的背景或词汇要求做额外说明
- 适合限制词汇风格、时代背景、术语体系
- 例如强制使用古代术语、行业术语、特殊称呼

### Persona Description

- 描述用户身份或用户设定
- 让模型更明确“用户在故事里是什么样的人”
- 和基础提示词中的“用户身份”相关，但这里更细致

### Char Description

- 角色的外貌、穿着、物理特征
- 偏视觉设定和外在描写

### Char Personality

- 对角色性格做更深层展开
- 比基础“性格特点”更适合写复杂心理、处事方式、待人模式

### Scenario

- 进一步定义当前场景
- 和基础“情景设定”相关，但这里常用于更细颗粒度的环境描述

### Enhance Definitions

- 用于强化特定行为定义
- 适合写”在什么情况下必须如何表现”
- 是让角色在关键情境下不跑偏的重要位置

### NSFW 内容控制

- 内置 identifier：`nsfw`
- 用于控制成人/敏感内容的处理策略
- 可以在 PresetEntry 中通过 `nsfw` identifier 注入 NSFW 相关的提示词或限制
- `safetySettings` 在包体字符串中存在，但不要默认写成“预设条目可直接控制提供商安全设置”。预设主要是提示词层控制；提供商侧安全策略仍以端点/模型实际支持为准

### Jailbreak 防越狱

- 内置 identifier：`jailbreak`
- 用于降低角色被用户通过提示词注入“越狱”（打破角色设定）的概率
- `charJailbreak` 宏（`{{charJailbreak}}`）：引用角色卡中设置的防越狱提示词
- `charInstruction` 宏（`{{charInstruction}}`）：引用角色指令，常与 jailbreak 配合
- 典型用法：在预设的 `jailbreak` entry 中写入角色保护规则

这仍然是提示词约束，不是安全沙箱，也不能保证模型一定不跑偏。

### Auxiliary Prompt

- 辅助提示
- 可以补充对话风格、行为细节、互动语气、补充限制
- 通常不承担主规则，而是做细修

### World Info (After)

- 用于在对话推进后补充世界或角色变化
- 适合写事件发生后的情绪变化、局势变化、阶段性状态更新

### Chat Examples

- 放更多对话示例
- 让模型学习稳定的台词风格、叙事方式和互动模板

### Chat History

- 告诉模型要如何理解和利用历史对话
- 核心作用是让上下文更连贯

### Post-History Instructions

- 历史记录之后的输出控制指令
- 常用来限制篇幅、排版、格式、节奏、写法
- 是很适合做“最终输出约束”的位置

## 5. 实际使用时的推荐分层

比较稳的写法是按下面的职责拆开：

- `Main Prompt`：最高优先级的总体规则
- `用户身份 / Persona Description`：用户在故事里的身份
- `角色设定 / Char Description / Char Personality`：角色是谁、长什么样、怎么做事
- `情景设定 / Scenario`：当前发生在哪里、处于什么局面
- `新示例聊天 / Chat Examples`：具体怎么说话
- `继续推进 / Group Chat / AI 帮答`：特殊对话场景
- `Post-History Instructions`：最终输出格式与长度限制

## 6. 和其他功能的关系

### 和世界书的关系

- 预设负责“系统级规则与表达方式”
- 世界书负责“客观世界知识与条件触发内容”
- 不要把大量世界观硬塞进预设，容易让主提示词过重

### 和宏的关系

- 预设里可以用宏注入动态信息
- 常见写法包括 `{{user}}`、`{{char}}`、变量宏、时间宏
- 对做状态驱动型玩法非常关键

### 和正则的关系

- 预设负责生成规则
- 正则负责生成前后或显示时的文本加工
- 两者适合配合而不是互相替代

## 7. 适合后续开发/配置时记住的点

- 预设是“行为模板”，不是单纯的备注栏
- 示例对话对模型风格稳定性很重要
- `Post-History Instructions` 很适合做篇幅、格式、语气限制
- 群聊和单聊最好拆开写不同推进逻辑
- 如果角色经常跑偏，优先检查 `Main Prompt`、`Enhance Definitions`、示例对话是否足够明确
