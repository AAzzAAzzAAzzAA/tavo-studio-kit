# AI 废词黑名单与文本清洗

中文 AI 模型在角色卡写作场景下频繁产出的套话、废词和过度文学化表达。生成角色卡内容后应做一轮清洗，删除或替换这些词。

灵感来源：社区 xkST 角色卡工坊项目的实战积累。

---

## 黑名单词库

### 情绪废词

"不禁"、"情不自禁"、"不由自主"、"鬼使神差"、"油然而生"、"心中涌起"、"一股暖流"、"下意识"

这些词的问题：它们让角色的情绪反应变得被动和模糊。"不禁笑了"不如直接写笑的动作和原因。

### 过度比喻

"宛如"、"仿佛"、"犹如"、"如同"、"好似"

这些词的问题：真正好的描写让读者自己感受到相似性，不需要每次都点明"像什么"。偶尔用可以，高频出现就是注水。

### 空洞修饰

"淡淡的"、"缓缓地"、"轻轻地"、"悄然"、"微微一笑"、"嘴角上扬"、"嘴角微微上扬"、"深邃的眼眸"

这些词的问题：它们是 AI 写作最常见的"万能修饰词"，用多了所有角色都像同一个人。

### 惊讶废词

"竟然"、"居然"、"没想到"

这些词的问题：高频出现时会让叙事节奏变得做作，仿佛每件事都出人意料。

### 文艺腔滥用

"氤氲"、"旖旎"、"悸动"、"涟漪"、"交织"、"命运的齿轮"

这些词的问题：本身不是坏词，但 AI 会在任何场景里不分青红皂白地使用，失去了原有的文学意义。

### 时间废标记

"那一刻"、"这一瞬间"

这些词的问题：几乎每段都出现时，就不再有任何强调效果。

### 表达回避

"难以言喻"、"无法形容"、"说不出的"

这些词的问题：角色卡的职责就是描述和形容，写"无法形容"等于交了白卷。

## 清洗规则

### 生成后清洗

对 AI 生成的 `description`、`personality`、`scenario`、`first_mes`、`mes_example`、`system_prompt`、`post_history_instructions` 以及世界书 `content` 做以下处理：

1. 删除黑名单中的词
2. 修复删除后的标点冗余（连续句号 `。。` → `。`，连续逗号 `，，` → `，`）
3. 修复删除后的多余空行（3 个及以上连续换行 → 2 个换行）
4. 修复删除后的多余空格（2 个及以上连续空格 → 1 个空格）

### 生成时预防

在生成 prompt 中加入提醒效果更好。示例：

```
以下词语禁止使用：不禁、油然而生、宛如、淡淡的、微微一笑、嘴角上扬、深邃的眼眸、命运的齿轮、氤氲、旖旎、悸动、涟漪、难以言喻。
用具体的动作、语气变化和细节代替这些空泛表达。
```

### 自定义扩展

黑名单应可被用户自定义。不同用户对"AI 味"的敏感度不同，有些人可能觉得"悸动"在特定语境下可以接受。建议：

- 提供默认黑名单作为起点
- 允许用户增删词条
- 在清洗前确认，而不是静默删除

## 替代写法建议

| 废词 | 问题 | 替代方向 |
|------|------|----------|
| 不禁笑了 | 被动、模糊 | 直接写笑的样子和原因 |
| 心中涌起一股暖流 | 套话 | 写具体的身体反应或行为变化 |
| 宛如蝴蝶般 | 过度比喻 | 直接写动作本身的质感 |
| 淡淡的笑 | 万能修饰 | 写这个笑具体长什么样、为什么笑 |
| 深邃的眼眸 | 任何角色都能套 | 写这双眼睛在这个场景里的具体状态 |
| 命运的齿轮开始转动 | 文艺腔 | 直接写发生了什么事 |
| 难以言喻的感觉 | 交白卷 | 既然要写，就必须言喻 |
