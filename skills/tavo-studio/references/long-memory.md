# Tavo 长记忆总结

## 1. 这是什么

长记忆是 Tavo 用来跨对话保留重要信息的能力。它的核心目的，是让 AI 不会在一次聊天结束后把与你有关的重要信息全部忘掉。

文档给出的直观理解是：

- 开启后，AI 像“有记性”的朋友
- 不开启时，AI 每次对话都像重新认识你

## 2. 开启与不开启的区别

### 开启长记忆

- Tavo 会把你提过的偏好、兴趣、习惯和重要细节保存为长期记忆，供后续聊天引用
- 这些信息可以在未来聊天中被再次引用
- 适合长期陪伴、长期养成和持续剧情型玩法

文档示例：

- 你说自己喜欢跑步
- 几个月后 AI 仍可能在上下文里使用这条记忆，例如问你最近有没有跑步

不要把长记忆写成确定承诺。它提供的是“保存并注入/引用的机会”，模型是否主动提及、提及是否准确，仍取决于当前上下文、端点模型、提示词和记忆是否启用。

### 不开启长记忆

- 每次对话都近似从零开始
- 你之前说过的偏好和事实不会自动被保留
- 适合一次性会话或不希望角色长期记住内容的场景

## 3. 长记忆管理是什么

长记忆管理的重点不是“AI 会不会记”，而是“你能不能控制记什么、什么时候记、怎么改”。

也就是说，这个功能包含两个层面：

- 是否启用长期记忆
- 当前记忆列表里到底保存了哪些内容

## 4. 两种记忆提取方式

文档重点讲了两种机制：

## 4.1 手动提取记忆

- 由用户主动决定什么时候提取
- 适合保存特别重要、特别明确、希望长期保留的信息
- 本质上是“这段对话很重要，请记住”

适合的内容：

- 用户稳定偏好
- 关系变化
- 关键剧情结论
- 必须长期保留的人设事实

## 4.2 10 条后自动提取记忆

- 系统在达到 10 条对话后可自动筛选有价值内容
- 用户不需要手动触发
- 更像“半自动记忆整理”

自动提取不是确定性的资料库写入。是否提取、提取质量和提取内容，仍依赖功能开关、模型请求是否成功以及系统判断。

适合的内容：

- 对话中多次出现的兴趣偏好
- 明显反复提到的信息
- 可能对后续长期互动有帮助的事实

## 5. 两种方式的区别

### 手动提取

- 用户主动控制
- 精准度高
- 适合保存“必须记住”的东西

### 10 条后自动提取

- 系统自动判断
- 省事
- 更适合长期聊天中的自然积累

一个很实用的理解方式是：

- 手动提取 = 明确收藏
- 自动提取 = 系统帮你做摘要和归档

## 6. 使用建议

### 什么时候优先手动提取

- 世界观或关系发生关键变化时
- 用户明确表达长期偏好时
- 你不希望模型以后忘掉某个设定时

### 什么时候让自动提取发挥作用

- 日常陪伴聊天
- 轻量长期关系养成
- 不想频繁手动管理记忆时

### 最稳的做法

两者并用：

- 自动提取负责“日常积累”
- 手动提取负责“关键节点”

## 7. 这个功能不等于世界书

边界一定要分清：

- 世界书保存的是世界设定、规则、剧情资料
- 长记忆保存的是聊天中提炼出的长期信息

如果你要保存的是：

- 世界法则
- 地点设定
- 历史背景

优先用世界书。

如果你要保存的是：

- 用户偏好
- 双方关系状态
- 会话中确认过的重要事实

优先用长记忆。

## 8. JavaScript API 可以怎么操作

JS API 文档补充了长记忆的脚本接口，后续做自动化时很有用。

### 获取当前记忆

- `await tavo.memory.current()`
- 返回当前聊天的记忆对象

对象主要字段：

- `id`
- `enabled`
- `memories`，类型是字符串数组

### 更新记忆

- `await tavo.memory.update(memory)`
- 可更新字段只有：
- `enabled`
- `memories`

这意味着你可以做：

- 脚本化开关记忆
- 批量替换当前聊天的长期记忆条目
- 用脚本把某些结论整理后写入记忆

注意：`tavo.memory.update(memory)` 是直接更新当前记忆对象，不等于触发官方的“手动提取/自动提取”流程，也不能保证运行摘要器或筛选器。

## 9. 适合后续开发/配置时记住的点

- 长记忆的核心价值是“跨会话保留重要信息”
- 手动提取适合关键节点，自动提取适合长期陪伴
- 它不是世界书，也不是预设
- 如果你要做自动化，可以直接操作 `enabled` 和 `memories`
